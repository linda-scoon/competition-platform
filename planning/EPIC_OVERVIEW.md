# Epic Overview

## Epic ID
EPIC-001

## Epic Name
MVP Buildout — Exercise Competition Platform

## Epic Goal
Implement the approved MVP exactly as specified in Docs 1–7, in small reviewable packets, with one PR per packet.

## Epic Rules
- One packet per PR.
- No packet may exceed one major system concern.
- No guessing.
- Respect dependencies.
- Use reviewable slices.
- Stop and ask if blocked by ambiguity.

## Human owner tasks
The human owner must:
- create the Epic in the project tool
- create child issues from `planning/IMPLEMENTATION_PACKETS.md`
- attach PRs to the Epic
- review and approve/merge in order
